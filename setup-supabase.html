Step 2: Check Table Status

Check if the survey_responses table already exists.

Step 3: Create Table (If Needed)

If the table doesn't exist, copy this SQL code and run it in Supabase SQL Editor:

-- Create survey responses table CREATE TABLE IF NOT EXISTS survey_responses ( id UUID DEFAULT gen_random_uuid() PRIMARY KEY, created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW() NOT NULL, vardas TEXT NOT NULL, el_pastas TEXT NOT NULL, telefonas TEXT NOT NULL, miestas TEXT, komentaras TEXT, type TEXT NOT NULL CHECK (type IN ('lauko', 'vidaus', 'kubila')), lauko_data JSONB, kubilas_data JSONB, vidaus_data JSONB ); CREATE INDEX IF NOT EXISTS idx_survey_type ON survey_responses(type); CREATE INDEX IF NOT EXISTS idx_survey_created_at ON survey_responses(created_at DESC); CREATE INDEX IF NOT EXISTS idx_survey_email ON survey_responses(el_pastas); ALTER TABLE survey_responses ENABLE ROW LEVEL SECURITY; CREATE POLICY "Allow anonymous survey submissions" ON survey_responses FOR INSERT TO anon WITH CHECK (true); CREATE POLICY "Authenticated users can view responses" ON survey_responses FOR SELECT TO authenticated USING (true);

Instructions:
1. Click "Copy SQL Code" above
2. Go to Supabase Dashboard
3. Open SQL Editor (left sidebar)
4. Click New query
5. Paste the SQL code
6. Click Run
7. Come back here and click "Check Table" again
